공통 가이드

DataBada는 고객사가 API 방식으로 제공된 데이터를 활용하여 다양한 업무를 개발하실 수 있도록 지원합니다.
API 서비스는 아래의 절차로 진행됩니다.

02

|

서비스 프로세스

계약 체결

APP Key발급

API 명세서 제공

테스트 진행 (테스트/운영)

Step 1

계약 체결

  • API 서비스 이용을 위해서는 계약 체결이 필수입니다. API 서비스 이용을 원하실 경우 문의를 남겨주시면 담당자가 빠르게 안내를 드리도록 하겠습니다.
  • 계약의 내용은 상품에 따라 상이하며, 이 때 인증키(APP Key) 발급과 관련한 추가 정보 및 신청 정보를 고객사에게 요청 드릴 수 있습니다.

Step 2

인증키(APP Key) 발급 및 IP White list 관리

  • 인증키 또는 APP Key란 API 이용자가 API 서비스의 이용 허가를 받은 주체임을 식별하 수 있도록 DataBada에서 이용자에게 개별적으로 부여하는 고유한 값을 말합니다.
  • 체결된 계약에 근거하여 DataBada에서 인증키를 발행하며, 이메일 또는 DataBada 홈페이지를 통해서도 확인하실 수 있습니다.
    ※ 개발(테스트)용, 운영용 별도 제공
  • API접근을 위해 고객사에서 사용하시는 서버 IP 최대 3개를 DataBada 담당자에게 전달해 주시면 White List에 등록해 드립니다.

Step 3

API 명세서 제공

  • API 운영을 위한 요청방식, 요청메시지, 응답메시지에 대한 상세 명세서를 제공합니다. 이 때 인증키를 포함하여 제공할 수 있으며 기관 인증 및 거래 무결성 검증 등의 중요 용도로 사용되는 중요 정보로 절대 외부에 노출되면 안됩니다.

Step 4

테스트 진행

  • 개발(테스트) 서버에서 연동 서비스 개발 후 서비스 점검이 완료되어야 운영(온라인) 서버 호출이 가능합니다.
  • 테스트 및 운영 오픈 일정은 DataBada 담당자와 협의를 통해 지정이 가능합니다.

※

본 문서의 인증 가이드가 절대적인 기준은 아니며, 이용하려는 개별 서비스나 관련 감독기관의 규정 변경에 따라 그 내용이 달라질 수 있습니다. 관련 규정 및 이용 서비스에 따라 변경 발생 시, 연동 개발 전 세부 항목에 대한 별도 협의가 진행될 수 있습니다.
